🟣Rider on Linux — the stable wins

Pin your toolkit on 2026.1 (niri) do stability

2026.1's launcher passes -Dawt.toolkit.name=auto and probes for Wayland — so on niri you're likely on native Wayland now. It's sharper on HiDPI but JetBrains documents popup/window-position/decoration bugs that hit niri's WM-controlled model hardest. Decide and pin in Help → Edit Custom VM Options:
• Glitches? Force -Dawt.toolkit.name=XToolkit (the proven XWayland path) and export _JAVA_AWT_WM_NONREPARENTING=1 for the gray-window fix.
• Native works fine? Pin -Dawt.toolkit.name=WLToolkit so it can't silently flip, and turn off synced UI/scale settings (they override your display config and re-introduce blur).

Source: 2026.1 Wayland · non-reparenting fix · scaling

track rider.vmoptions (with your chosen toolkit) in a rider stow package; add the env var if you go XToolkit

Heap + solution-wide-analysis tuning do performance

The biggest day-to-day difference on a NopCommerce-scale solution. Help → Edit Custom VM Options: set -Xmx to 4–8 GB. Settings → Editor → Inspection Settings: exclude Roslyn analyzers from solution-wide analysis (JetBrains explicitly recommends this for large solutions), and scope/skip generated + obj output. Watch the separate backend process — that's where the RAM goes.

Source: heap · SWA · 2026.2 EAP rebatched Roslyn comms for 2–3× faster branch switching (relevant to your worktree cadence — but EAP)

track a rider.vmoptions in dotfiles

Pin the toolset to your mise SDK do mise

Known sharp edge: Rider often prioritizes /usr/bin/dotnet over the mise-injected env even when MSBuild sees the right SDK. In Settings → Build, Execution, Deployment → Toolset and Build, set the .NET CLI path explicitly to the resolved mise install (and choose custom runtime, not auto). Re-point it after every mise SDK bump (or symlink a stable current path mise updates).

Source: Toolset docs · Rider+mise report

Reproducible Rider config (Stow-friendly) do dotfiles

Rider config isn't plain text, so go export-based, not symlink-based. File → Manage IDE Settings → Export Settings → commit the settings.zip. Additionally Stow the keymap XML from ~/.config/JetBrains/Rider*/keymaps/ (single safe file) and track *.sln.DotSettings + .idea/* except workspace.xml (machine state — never track). Prefer this over account-based Settings Sync, which isn't a VCS artifact you control.

Source: Sharing IDE settings

🟢Terminal layer — editor-agnostic (still fully applies)

zsh-abbr do zsh

Abbreviations expand to the real command before running, so history/atuin store git checkout, not gco — clean Ctrl-R reuse across your ~50 environments. Use abbrs for things you want to see expand; keep true aliases only for flag-bundles (ls→eza …).

Source: olets/zsh-abbr

add a sourced abbreviations.zsh + convert the alias list

lazygit dual-diff (delta + difftastic) do git

Even with Rider's Git UI, terminal lazygit is great for quick staging. Delta as the pager for review; difftastic as external diff for structural AST diffs (big C# refactors). difft can't be a pager — wire it behind git dft, leaving plain git diff on delta.

Source: lazygit pagers · difft git

lazygit config + external-diff binding

zellij: autolock + zjstatus + per-project layout do zellij

zellij-autolock passes keys straight through when a pane runs fzf/lazygit (and nvim if you ever use it) — smoother modal pickers. zjstatus = themeable status bar showing session/branch/per-pane command, so you can tell which tab holds which agent. And scaffold a real per-NopCommerce KDL (your zp already prefers layouts/<project>.kdl): tabs for shell, dotnet watch, lazygit, and N agent tabs.

Source: autolock · zjstatus / awesome-zellij

all three: plugin blocks in config.kdl + layouts/nxg-csharp-nopcommerce.kdl

watchexec + just test loops do .NET loop

Codify terminal test loops as just recipes driven by watchexec (gitignore-aware, debounced): watchexec -e cs -i '**/obj/**' -- dotnet test --filter 'FullyQualifiedName~{Class}'. Complements Rider's runner for fast headless feedback; reproducible across envs. Watch quirk: avoid project folders containing a . (breaks hot-reload on Linux).

🔴Skip / be skeptical

Leaving Rider's toolkit on auto skip

On 2026.1/niri, auto silently lands you on native Wayland with its popup/positioning bugs. Pin one toolkit explicitly (see the Rider section) so behavior is deterministic and reproducible.

Nushell / fish · zinit · Electron agent GUIs skip

Full rewrites of a lean zsh that already starts fast; GUI agent dashboards break your Wayland/keyboard-first ergonomics. Stay terminal-native.
